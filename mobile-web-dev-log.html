Portfolio Production Flow의 Step.1에서는 가상의 4인조 퍼포먼스 그룹 AURORA:NE와 팬덤 Orbit을 주제로 선정했습니다. 실제 아이돌 이미지에 의존하지 않고, 자체 팬덤 세계관과 기능 구조를 만들어 저작권 리스크 없이 포트폴리오로 설명 가능한 방향을 선택했습니다.

  • 새 프로젝트 폴더 생성 fandom-mobile-web-project
  • 1-17단계 전체 과정 문서화: 아이템 선정, IA, 와이어프레임, 디자인, 구현, 배포, 케이스스터디
  • React/TypeScript 소스 작성: AppShell, Feed, Vote, Chat, Member, Live 기능 구조
  • 채팅창 구현: Chat List, Chat Room, Chat Bubble, Chat Input, Message Status, Safety UX
  • 검증용 정적 결과물 생성 dist/index.html 기준으로 로컬 QA 완료

Step.1 산출물

  • 1-17단계 프로세스 문서
  • React/TypeScript 구현 소스
  • 팬덤 데이터 타입 설계
  • QA 결과 문서
  • 배포 정보 문서

검증 기록

Step.1 결과물이 실제로 확인 가능한 수준인지 로컬 파일과 배포 URL에서 나눠 검증했습니다. 아래 표는 포트폴리오에 넣을 수 있도록 검증 목적, 확인 기준, 최종 결과를 분리해 정리한 QA 기록입니다.

배포 확인 주소 https://dist-flame-mu.vercel.app Vercel Production 배포 후 접속 상태와 주요 인터랙션을 확인했습니다.
검증한 것 확인 기준 결과
배포 접속 배포 URL을 열었을 때 페이지가 정상 로드되는가 통과 200 OK
모바일 프레임 팬덤앱형 모바일웹 기준인 430px 이하 화면에서 동작하는가 통과 430px 프레임 확인
탭 이동 Home, Vote, Chat 등 주요 탭을 눌렀을 때 화면이 전환되는가 통과 주요 탭 이동 확인
투표 UX Vote 탭에서 선택 후 진행률과 참여 상태가 바뀌는가 통과 선택 후 진행률 변경
채팅 UX 채팅방, 말풍선, 입력창, 메시지 상태, 안전 안내가 포함되는가 통과 팬덤앱 채팅 구조 구현
레이아웃 안정성 모바일 화면에서 가로 스크롤이나 화면 밖으로 튀는 요소가 없는가 통과 가로 오버플로 없음

Step.2 지금부터의 디벨롭

Step.2부터는 현재 배포된 결과물을 기준으로 사용자가 확인한 뒤, 실제 포트폴리오 완성도를 더 높이기 위한 개선을 누적합니다. 이 페이지는 앞으로 그 여정을 계속 기록하는 기준 페이지입니다.

Step.2에서 기록할 것

  • 사용자가 배포 URL을 보고 남긴 피드백
  • 디자인 밀도, 화면 구성, 카피, 인터랙션에 대한 개선 결정
  • 채팅창 UX의 다음 상태: 입력 중, 실패, 신고/차단 플로우, 이미지 메시지
  • React 컴포넌트 리팩토링과 TypeScript 타입 보강
  • 배포 URL 재검증 결과

다음 개선 로드맵

Step.1은 “가이드가 실제 결과물로 이어지는가”를 증명한 단계입니다. 다음은 최종 포트폴리오 제출물처럼 보이도록 화면 수, 상태 수, 케이스스터디 설득력을 더 높이는 단계입니다.

  1. Home 화면의 비주얼 완성도 강화: 실제 앨범/라이브 느낌의 그래픽 레이어 추가
  2. Members 상세 화면 추가: 멤버별 팬 피드, 대표 영상, 응원 CTA
  3. Chat UX 고도화: typing 상태, failed 상태, 이미지/이모지 메시지, 신고/차단 모달
  4. Vote UX 고도화: 투표 완료 축하 모션, 랭킹 변화, 참여 배지
  5. 케이스스터디 페이지 추가: Problem, IA, Design System, React Build, QA Result
  6. Vercel 재배포 및 QA 로그 업데이트
