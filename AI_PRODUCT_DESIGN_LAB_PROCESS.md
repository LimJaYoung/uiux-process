# AI Product Design Lab 제작 과정 교안

## 0. 프로젝트 개요

**사이트 이름**
AI Product Design Lab

**사이트 목표**
UIUX·프로덕트 디자이너가 AI를 활용해 기획, 설계, 디자인, 코딩, 배포까지 완주하고 고퀄리티 포트폴리오를 만드는 코스형 학습 사이트를 제작한다.

**핵심 문장**
AI로 기획부터 배포까지 완주하는 포트폴리오 코스

**대상 사용자**
- UIUX 디자이너
- 프로덕트 디자이너
- 포트폴리오를 만들어야 하는 취업·이직 준비자
- AI 도구를 활용해 직접 프로덕트를 만들고 싶은 디자이너

**최종 결과물**
- 정적 웹사이트
- Vercel 배포 URL
- Figma 구조화 파일
- 자동화 확장을 위한 JSON / GitHub Actions 기본 구조

## 6. 가로 스크롤 포트폴리오 레일 구현

Awesomic의 서비스 카드 섹션처럼 수료생 포트폴리오를 보여주는 가로 레일을 구현했다.

### 6-1. 구현 중 발생한 문제

초기에는 다음 문제가 있었다.

- 부모 슬라이드가 통째로 움직임
- 아래 가로 스크롤바가 생김
- 마우스휠 이동 거리가 너무 짧음
- 슬라이드가 다른 섹션 위로 떠다님
- 마지막 지점에서 슬라이드가 튕김

### 6-2. 최종 해결 방향

최종 구조:
- `service-strip`이 스크롤 구간을 확보한다.
- `service-sticky`가 화면 안에서 고정된다.
- `service-track`이 스크롤 진행도에 따라 가로로 이동한다.
- 레일 끝에 도달하면 다음 섹션으로 넘어간다.
- 가로 스크롤바는 노출하지 않는다.

### 6-3. 구현 의도

이 레일은 단순 슬라이더가 아니라 수료생 포트폴리오를 시각적으로 보여주는 핵심 쇼케이스 영역이다.

## 13. 자동화 큐레이션 구조 설계

AI + 디자인 뉴스와 YouTube 콘텐츠를 자동화로 수집할 수 있는 구조를 설계했다.

### 13-1. 자동화 가능 여부

가능하다.

정적 사이트라도 다음 구조를 사용하면 매일 콘텐츠를 갱신할 수 있다.

1. GitHub Actions가 매일 실행된다.
2. RSS, 공식 블로그, YouTube RSS/API에서 데이터를 수집한다.
3. JSON 파일을 업데이트한다.
4. 변경된 JSON이 커밋된다.
5. Vercel이 자동으로 다시 배포한다.

### 13-2. 추가한 파일

자동화 확장을 위해 다음 파일을 추가했다.

- `.github/workflows/daily-content.yml`
- `scripts/collect-content.mjs`
- `data/ai-design-news.json`
- `data/youtube-videos.json`

### 13-3. 운영 기준

운영 기준:
- API 키는 프론트엔드에 넣지 않는다.
- API 키는 GitHub Secrets에 보관한다.
- 기사 전문을 복사하지 않는다.
- 제목, 출처, 링크, 짧은 요약만 제공한다.
- YouTube 영상도 임베드보다 큐레이션 목록 중심으로 시작한다.

## 17. 교안에서 강조할 핵심 메시지

이 사이트는 단순히 AI 도구 사용법을 알려주는 사이트가 아니다.

핵심은 다음이다.

> UIUX·프로덕트 디자이너가 AI를 활용해 아이디어를 실제 작동하는 결과물로 만들고, 그것을 포트폴리오로 정리해 배포까지 완주하는 과정

따라서 교안에서는 특정 도구 사용법보다 다음 흐름을 강조해야 한다.

- 문제를 정의한다.
- 사용자를 정한다.
- MVP 범위를 정한다.
- 화면 흐름과 IA를 설계한다.
- UI 방향과 디자인 시스템을 잡는다.
- AI 도구로 작동하는 화면을 구현한다.
- 브라우저에서 QA한다.
- Vercel로 배포한다.
- 제작 과정을 케이스 스터디로 정리한다.
- 필요하면 뉴스/영상 큐레이션까지 자동화한다.
