Beginner

초급: 간단한 MVP 포트폴리오

AI 코딩이 낯선 디자이너가 작은 서비스 아이디어를 배포까지 완주합니다.

기획 15% → 설계 30% → 디자인 50% → 코딩 70% → 배포 100%
Intermediate

중급: 서비스형 프로덕트 포트폴리오

문제정의 → UX 설계 → UI 시스템 → 프론트 구현 → 케이스 스터디
Advanced

고급: 자동화 기반 실무형 포트폴리오

하네스와 n8n 자동화까지 연결해 팀프로젝트와 브랜딩 포트폴리오를 고도화합니다.

전략 → 인터랙션 → 자동화 → 퍼블리싱 → 발표자료

How It Works

명확한 프롬프트로 기획부터 배포까지 연결합니다

막연한 AI 요청이 아니라 목적, 입력값, 출력 형식, 검수 기준을 갖춘 프롬프트로 포트폴리오 결과물을 단계별로 만듭니다.

01

사이트에서 학습

단계별 목표, 준비물, 프롬프트, 체크리스트를 확인합니다.

02

AI 도구에서 실행

Codex, AI 코딩 도구, 디자인 도구에 맞춰 프롬프트를 실행합니다.

03

브라우저에서 확인

프로토타입, 케이스 스터디, 배포 URL로 결과를 직접 봅니다.

04

체크리스트로 개선

UX, UI, 접근성, 반응형 기준으로 다시 요청하고 다듬습니다.

Workflow

포트폴리오 완주 로드맵

진단에서 퍼스널 브랜딩까지, 하나의 포트폴리오를 끝까지 만드는 순서입니다.

진단내 시작점 찾기 기획문제와 MVP 정의 설계흐름과 화면 목록 디자인UI 기준 정리 코딩작동 화면 구현 케이스QA와 스토리 정리 배포공유 URL 만들기 자동화규칙 저장

FAQ

자주 묻는 질문

5분 오리엔테이션
코드를 전혀 몰라도 시작할 수 있나요?

처음에는 HTML, CSS, JavaScript 파일을 열고 AI 코딩 도구와 함께 수정하는 방식으로 시작합니다.

Figma 디자인을 바로 개발로 바꾸는 사이트인가요?

그보다 앞단의 기획, 설계, 디자인 기준까지 함께 정리해서 구현 가능한 요청으로 바꾸는 흐름입니다.

최종 결과물은 무엇인가요?

브라우저에서 열리는 프로토타입, 배포 URL, 포트폴리오에 설명할 수 있는 제작 기록입니다.

자료실은 언제 쓰나요?
