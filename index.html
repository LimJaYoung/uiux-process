Bottleneck

디자이너가 Claude Code에서 막히는 지점을 먼저 해결합니다

기획이 흐릿하면 Claude의 결과물도 흐릿해집니다.

화면 구조가 없으면 구현 요청이 계속 흔들립니다.

검수 기준이 없으면 예쁜데 애매한 화면에서 멈춥니다.

배포 경험이 없으면 만든 결과물을 공유하기 어렵습니다.

70% 기획과 설계에서 먼저 방향을 고정합니다.
40% 구현 전 프롬프트를 화면 단위로 쪼갭니다.
80% 검수와 개선 요청을 반복 루틴으로 만듭니다.

FAQ

5분 오리엔테이션
코드를 전혀 몰라도 시작할 수 있나요?

처음에는 HTML, CSS, JavaScript 파일을 열고 Claude Code와 함께 수정하는 방식으로 시작합니다.

Figma 디자인을 바로 개발로 바꾸는 사이트인가요?

그보다 앞단의 기획, 설계, 디자인 기준까지 함께 정리해서 구현 가능한 요청으로 바꾸는 흐름입니다.

최종 결과물은 무엇인가요?

브라우저에서 열리는 프로토타입, 배포 URL, 포트폴리오에 설명할 수 있는 제작 기록입니다.

자료실은 언제 쓰나요?

프롬프트 템플릿, Claude Code 활용 가이드, QA 체크리스트가 필요할 때 바로 열어 씁니다.
