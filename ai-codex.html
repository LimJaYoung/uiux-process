언제 쓰나

디자인 기준은 있지만 실제 작동 화면이 없을 때 씁니다. HTML, CSS, JS를 수정하고 테스트와 QA까지 연결할 수 있습니다.

활용 방법

  • 기존 파일 구조를 먼저 읽게 합니다.
  • 화면 단위로 작게 구현을 요청합니다.
  • 모바일, hover, focus, 빈 상태를 함께 요청합니다.
  • 수정 파일과 검수 결과를 디자이너가 이해할 말로 설명하게 합니다.

바로 쓰는 프롬프트

당신은 UIUX 디자이너와 협업하는 시니어 프론트엔드 엔지니어입니다.
기존 파일 구조를 먼저 파악한 뒤, 아래 디자인 기준에 맞게 화면을 구현해주세요.

작업 기준:
1. 기존 스타일 규칙을 최대한 유지
2. 모바일과 데스크톱 모두 확인
3. 버튼, 링크, 카드의 hover와 focus 상태 포함
4. 텍스트가 버튼이나 카드 밖으로 넘치지 않게 처리
5. 수정한 파일과 확인 방법을 짧게 설명

디자인 기준:
[여기에 UI 방향을 붙여넣으세요]
