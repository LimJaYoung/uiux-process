  • 케이스스터디를 위한 UIUX 장표 제작: 테스트 결과와 최종 디자인을 포트폴리오 발표 자료로 정리

01. 아이템 선정

포트폴리오의 시작은 브랜드 선정입니다. 좋아하는 브랜드보다 결과물의 완성도를 높일 수 있는 브랜드를 선택해야 합니다. 리뉴얼 명분이 분명하고, 인터랙티브 웹으로 구현했을 때 시각적 임팩트가 큰 브랜드가 면접관의 시선을 더 끌지 않을까요?

  • Before & After의 결과가 명확할 수 있는가
  • 현재 웹사이트의 문제점이 명확해 리뉴얼 명분이 생기는가
  • 이미지, 영상, 공간, 제품, 라이프스타일 등 비주얼 소스가 풍부한가
  • 스크롤 애니메이션, 모션 타이포, 섹션 전환이 자연스럽게 어울리는가
  • 최상급 퀄리티의 레퍼런스 자료가 충분한가
너는 UIUX 기획자이자 포트폴리오 디렉터야.

취업용 UIUX 풀스택 포트폴리오를 제작하기 위해
K-브랜드 웹사이트 리뉴얼 반응형 웹 포트폴리오를 만들려고 해.

목표:
- SaaS형 서비스가 아니라 브랜드 경험 중심의 인터랙티브 웹을 제작
- Figma 디자인과 HTML/CSS/JavaScript 구현까지 연결
- 면접관이 첫 화면에서 시각적 임팩트를 느낄 수 있는 결과물 만들기

브랜드 후보를 20개 추천해줘.
반드시 K-브랜드 중심으로 추천하고, 쇼핑몰 스타일이 아닌 브랜드 경험/공간/라이프스타일/서비스형 브랜드를 우선해줘.

평가 기준:
1. Before & After 결과의 명확성
2. 현재 웹사이트의 리뉴얼 명분
3. 비주얼 임팩트 가능성
4. 인터랙티브 구현 여지
5. 개인 포트폴리오로 구현 가능한 복잡도
6. 최상급 퀄리티의 레퍼런스 자료 충분성

출력 형식:

06. 참고 레퍼런스 제시

이 단계에서는 바로 디자인 시스템을 추출하지 않습니다. 먼저 리뉴얼 방향에 맞는 최상급 레퍼런스 후보를 여러 개 제시하고, 사용자가 그중 하나를 기준 레퍼런스로 선택합니다. 선택이 끝난 뒤 다음 단계에서 해당 사이트의 폰트, 여백, 그리드, 이미지 비율을 정밀 추출합니다.

  • 기준 레퍼런스 후보 5~8개를 먼저 제시한다.
  • 각 후보의 레이아웃, 타이포, 이미지, 모션, 구현 난이도를 비교한다.
  • 사용자가 후보 중 1개를 선택할 수 있도록 장단점을 명확히 정리한다.
  • 선택된 1개 사이트만 다음 단계에서 디자인 시스템 추출 대상으로 사용한다.

사용자는 제시된 후보를 검토한 후 최종 기준 레퍼런스 1개를 직접 선정합니다. 선정된 기준 레퍼런스는 프로젝트의 전체 레이아웃, 타이포그래피, 그리드 시스템, 콘텐츠 구성의 기준이 되며, 이후 디자인 방향성 역시 이를 중심으로 전개합니다.

기준 레퍼런스 선정은 프로젝트의 최종 비주얼 퀄리티를 결정하는 핵심 과정입니다. 따라서 사용자가 원하는 방향성과 수준에 부합하는 레퍼런스를 찾을 때까지 후보 탐색, 비교, 교체 과정을 반복적으로 진행합니다. 충분한 검토 없이 레퍼런스를 확정하지 않으며, 사용자가 납득할 수 있는 기준과 근거를 바탕으로 최종 레퍼런스를 선정합니다.

서브 레퍼런스는 기본적으로 제공하지 않으며, 사용자가 추가 요청한 경우에만 히어로 섹션, 카드 UI, 모션, CTA, 인터랙션 등 특정 요소에 한하여 별도로 제안합니다. 단, 서브 레퍼런스는 부분 요소 참고용으로만 활용하며 전체 디자인 구조의 기준은 항상 기준 레퍼런스를 유지합니다.

07. 레퍼런스 디자인 시스템 추출

사용자가 선택한 기준 레퍼런스 1개를 기준으로 실제 디자인 시스템을 추출합니다. 이 단계에서 추출한 폰트명, 폰트 두께, 자간, 줄간격, 폰트 크기, 여백, 그리드 값이 이후 Figma와 바이브코딩의 기준이 됩니다. 절대 감으로 추정하지 않고, 반드시 기준 레퍼런스에서 확인한 값을 기준으로 시스템을 정리합니다.

* 측정 근거가 없는 값은 디자인 시스템으로 확정하지 말고, 반드시 사용자가 추출 근거를 확인할 것

정확한 디자인 시스템 추출을 위해 반드시 필요한 것

디자인 시스템 추출은 감상문이 아니라 측정 작업입니다. AI는 레퍼런스 사이트를 눈으로 본 느낌만으로 폰트 크기, 여백, 그리드 값을 만들면 안 됩니다. 반드시 화면 캡처, 개발자도구, computed style, CSS 변수, DOM selector, 이미지 비율 측정 중 가능한 근거를 확보한 뒤 값과 단위를 기록해야 합니다.

확정 가능한 값의 조건
1. 어떤 화면/섹션/요소에서 측정했는지 적혀 있어야 함
2. px, %, rem, vw 등 단위가 있어야 함
3. selector, CSS rule, computed style, screenshot 측정 위치 중 하나 이상의 근거가 있어야 함
4. 확인하지 못한 값은 "추정"이 아니라 "확인 필요"로 남겨야 함
5. 추출값 검수표에서 "확정"으로 표시된 값만 08 이후 단계에 사용할 수 있음 

너는 선택된 기준 레퍼런스의 디자인 시스템을 실제 화면 기준으로 추출하는 UI 시스템 분석가야.

기준 레퍼런스 캡처는 사용자가 직접 하는 작업이 아니라, AI 에이전트가 먼저 해결해야 하는 작업입니다. AI는 Playwright, 브라우저 캡처, PDF 저장, 분할 캡처, 이미지 변환 등 가능한 방법을 순서대로 시도해 기준 레퍼런스 index 페이지를 캡처하고 Figma Reference Frame에 배치해야 합니다.

AI 우선 캡처 원칙
1. AI가 Playwright fullPage screenshot 또는 브라우저 자동화 캡처를 먼저 시도
2. 실패하면 AI가 브라우저 기본 캡처, PDF 저장, 분할 캡처, HTML 저장 후 로컬 캡처를 순서대로 시도
3. 그래도 어렵다면 AI가 가능한 확장 프로그램, 외부 캡처 도구, 온라인 캡처 서비스 사용 가능성을 확인
4. 설치, 로그인, 유료 서비스, 권한 승인이 필요한 경우에만 사용자에게 가능 여부를 묻기
5. 사용자가 직접 캡처 파일을 만들어 주는 것을 기본 전제로 하지 않기

캡처가 확보되지 않았다면 CSS만 보고 와이어프레임을 확정하지 않습니다. CSS/DOM 분석은 보조 근거로만 사용하고, 캡처 이미지나 PDF를 Figma Reference Frame에 배치한 뒤 진행합니다.
