기획부터 배포까지 완성하는 서비스형 포트폴리오

Intermediate에서 만들어야 할 것

이 레벨의 목표는 "작동하는 화면"이 아니라 "서비스를 설계하고 완성하는 과정"을 증명하는 것입니다. 기획의 깊이, 문제 정의의 논리, 설계의 일관성, 구현의 완성도, 배포와 QA까지 모두 포트폴리오 평가 대상이 됩니다.

  • 서비스 기획서: 타깃, 문제, 포지셔닝, 핵심 가치 제안
  • 리서치 요약: 경쟁 서비스, 사용자 시나리오, 기회 영역
  • PRD: 기능 우선순위, 요구사항, 성공 지표, 제외 범위
  • UX 설계서: IA, 핵심 플로우, 화면 명세, 상태 정의
  • 디자인 시스템: 토큰, 컴포넌트, Design.md, 레퍼런스 기준
  • 시안 시각화: 핵심 화면, 상태 화면, 플로우 검증용 목업
  • 개발 결과물: 다중 화면, 상태 화면, 반응형, 배포 URL
  • React 리팩토링: 컴포넌트 구조, props, state, route 설계
  • 인터랙티브 웹: 필터, 입력, 전환, 저장, 피드백, 마이크로인터랙션
  • 케이스 스터디: 의사결정, AI 활용 워크플로우, 결과와 회고

01. 기획 전략 수립

Intermediate 기획은 아이디어를 많이 뽑는 것보다, 포트폴리오에서 설명 가능한 문제를 고르는 데 집중합니다. 서비스는 최소 3개 이상의 화면과 명확한 사용자 여정을 가져야 하고, 기능보다 문제 해결 구조가 먼저 보여야 합니다.

  • 서비스 카테고리와 타깃 사용자 선정
  • 문제 상황, 사용 맥락, 기존 대안의 한계 정리
  • 핵심 가치 제안과 포트폴리오 차별점 정의

중급 프로젝트 아이템 검토

너는 시니어 프로덕트 매니저이자 UX 포트폴리오 멘토야.

내가 Intermediate 수준의 서비스형 포트폴리오를 만들려고 해.
아래 후보 아이디어를 중급 포트폴리오 기준으로 평가해줘.

후보 아이디어:
1. [아이디어 A]
2. [아이디어 B]
3. [아이디어 C]

평가 기준:
- 실제 사용자 문제가 명확한가?
- 리서치와 경쟁 분석으로 근거를 만들 수 있는가?
- 핵심 화면 8~12개로 서비스 흐름과 상태를 설계할 수 있는가?
- 디자인 시스템을 보여주기 좋은가?
- 개발 난이도가 2~4주 안에 가능한가?
- 케이스 스터디에서 의사결정을 설명하기 좋은가?

출력 형식:
| 아이디어 | 문제 명확성 | 설계 확장성 | 구현 현실성 | 포트폴리오 설명력 | 총평 |

03. 문제 정의와 솔루션 방향

문제 정의는 "무엇을 만들까"가 아니라 "왜 이 흐름이 필요한가"를 설명하는 기준입니다. 리서치 결과를 HMW, 사용자 여정, 핵심 가설로 압축한 뒤 솔루션의 방향을 정합니다.

  • HMW 문제 정의문
  • 사용자 여정의 핵심 마찰 지점
  • 솔루션 컨셉과 핵심 가설

HMW는 How Might We의 줄임말로, 리서치에서 발견한 문제를 "어떻게 하면 우리가 [사용자]가 [상황]에서 [목표]를 달성하도록 도울 수 있을까?" 형태의 기회 질문으로 바꾸는 방법입니다. 너무 넓으면 방향이 흐려지고, 너무 좁으면 바로 기능명이 되어버리므로 사용자, 상황, 목표가 함께 드러나야 합니다.

문제 정의서 작성

지금까지의 기획과 리서치 내용을 바탕으로 문제 정의서를 작성해줘.

기획 요약:
[서비스 포지셔닝 붙여넣기]

리서치 요약:
[경쟁 분석, 사용자 인사이트 붙여넣기]

문제 정의서는 아래 구조로 작성해줘:
1. Background: 왜 이 문제가 지금 중요한가
2. Target User: 누구의 문제인가
3. Current Behavior: 현재 사용자는 어떻게 해결하고 있나
4. Pain Point: 어디에서 불편이 생기나
5. Root Cause: 근본 원인은 무엇인가
6. HMW Statement: 어떻게 하면 [사용자]가 [상황]에서 [목표]를 달성하도록 도울 수 있을까?
7. Opportunity: 이 문제를 해결하면 어떤 가치가 생기나

포트폴리오 케이스 스터디에 그대로 넣을 수 있게 작성해줘.

  • 목표, 사용자, 문제, 핵심 가설

06. 디자인 시스템과 Design.md

디자인은 PRD와 UX 설계를 시각 언어로 바꾸는 단계입니다. 레퍼런스를 분석하고, 토큰과 컴포넌트 규칙을 만든 뒤 Design.md에 정리해야 AI가 일관된 화면을 확장할 수 있습니다.

  • 레퍼런스 3개 분석과 디자인 방향 결정
  • 컬러, 타이포, spacing, radius, shadow 토큰
  • 버튼, 카드, 인풋, 내비게이션 컴포넌트 규칙

5. 모바일 390px / 데스크톱 1920px 프레임 반응형
6. 데스크톱 본문 콘텐츠 최대 1440px 컨테이너 정렬
7. 접근성: 대비, 포커스, 키보드 탐색
8. 상태 화면: 빈 상태, 로딩, 에러, 성공 피드백
9. 포트폴리오에서 설명하기 약한 부분

결과는 ❌ 필수 수정 / ⚠️ 개선 권장 / ✅ 통과로 분류하고, 필수 수정은 수정 방법까지 써줘.
