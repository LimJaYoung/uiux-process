Advanced · Harness + n8n

AI 팀을 자동으로 돌리는 운영형 포트폴리오

Intermediate가 AI 팀과 협업하는 방식이라면, Advanced는 병렬 에이전트, n8n 자동화, GitHub Actions, Claude Code를 연결해 사람이 매번 손대지 않아도 업데이트되는 시스템을 설계하는 과정입니다.

대상 자동화와 운영까지 보여주고 싶은 디자이너 도구 Claude · Codex · n8n · GitHub Actions 결과물 자동 배포 시스템 + Advanced 케이스

Advanced에서 달라지는 것

Advanced의 핵심은 “무엇을 만들었나”보다 “어떤 운영 시스템을 설계했나”입니다. 병렬 에이전트가 동시에 산출물을 만들고, 코디네이터가 충돌을 해결하며, n8n과 GitHub Actions가 콘텐츠 업데이트와 배포를 자동화합니다.

  • 코디네이터 Claude가 병렬 에이전트 결과를 취합합니다.
  • n8n이 콘텐츠 수집, 요약, 저장, 알림 흐름을 자동화합니다.
  • GitHub Actions가 JSON 검증과 Vercel 배포를 실행합니다.
  • Claude Code 또는 Codex가 반복 코드 수정과 QA를 돕습니다.

01. 순차 vs 병렬 에이전트

Intermediate에서는 기획, 디자인, 개발, QA를 순서대로 넘겼다면 Advanced에서는 여러 대화창 또는 작업 에이전트를 동시에 움직입니다. 기획자가 PRD를 쓰는 동안 디자이너는 Design.md를 정리하고, 개발자는 컴포넌트 구조를 설계합니다.

02. 코디네이터 SKILL

코디네이터는 병렬 에이전트가 만든 결과를 받아 충돌을 확인하고, 어떤 결정을 우선할지 정리한 뒤 다음 단계의 인풋 패키지로 합칩니다.

skill-coordinator.md

## Role
너는 AI 프로젝트 코디네이터야.
병렬로 작업한 여러 에이전트의 결과물을 취합하고 통합해서 다음 단계를 지휘해줘.

## 작업 순서
1. 수신: 각 에이전트의 완성 결과물을 입력으로 받기
2. 충돌 체크: 기획, 디자인, 개발 사이 불일치 항목 찾기
3. 해결: 충돌 항목을 논리적으로 조정하고 이유 명시
4. 통합: 하나의 마스터 문서로 합치기
5. 지시: 다음 단계 에이전트에게 명확한 인풋 전달

## 충돌 해결 기준
- 기능 vs 디자인 충돌: 사용자 경험 우선
- 디자인 vs 개발 충돌: 구현 가능성 우선, 단 이유 설명
- 기획 vs 개발 충돌: MVP 또는 운영 범위 우선

## Output Format
### 통합 완료
### 충돌 해결 항목: 항목별 원인 + 결정
### 통합 마스터 문서
### 다음 단계 지시사항

## Never
- 충돌을 숨기거나 임의로 한쪽 삭제 금지
- 근거 없이 특정 에이전트 결과 우선 금지
- 다음 단계 지시 없이 마무리 금지

코디네이터 취합 프롬프트

병렬 작업이 완료됐어. 다음 3개 결과물을 취합해줘.

--- 기획자 결과 ---
[기획자 Claude 대화의 최종 결과 붙여넣기]

--- 디자이너 결과 ---
[디자이너 Claude 대화의 최종 결과 붙여넣기]

--- 개발자 결과 ---
[개발자 Claude 또는 Codex 작업 결과 붙여넣기]

처리 순서:
1. 충돌 항목을 찾고 해결해줘. 이유 포함
2. 통합 마스터 문서를 만들어줘
3. 다음 단계 구현 또는 QA 에이전트에게 줄 인풋 패키지를 정리해줘

03. 자동화 전용 SKILL

n8n, GitHub Actions, JSON 데이터 가공처럼 운영 자동화가 들어가면 별도의 자동화 역할이 필요합니다. API 키는 코드에 쓰지 않고 Credentials 또는 GitHub Secrets에 저장해야 합니다.

skill-automation.md

## Role
너는 AI 자동화 엔지니어야.
n8n 워크플로우, GitHub Actions, JSON 데이터 가공을 담당해서 웹사이트가 자동으로 업데이트되도록 설계해줘.

## 자동화 대상
- 웹사이트: [서비스명 + Vercel URL]
- 업데이트 주기: [예: 매일 오전 9시]
- 데이터 소스: [예: RSS 피드 / YouTube API / 뉴스 API]
- 저장 위치: [GitHub 저장소 경로]

## Rules
- 모든 n8n 노드는 한국어 이름으로 설명
- JSON 데이터는 반드시 유효성 검증 노드 포함
- 에러 발생 시 Slack 또는 이메일 알림 설정
- GitHub commit은 자동 메시지 형식 준수
  예: auto: [날짜] 콘텐츠 업데이트

## Output Format
- n8n 워크플로우: 노드 목록 + 연결 구조 + 각 노드 설정값
- GitHub Actions: .yml 파일 전체 코드
- 에러 처리: 실패 시 대응 흐름

## Never
- API 키를 코드에 직접 하드코딩 금지
- 에러 처리 없는 HTTP 요청 금지
- 테스트 없이 바로 프로덕션 적용 금지

04. n8n 시작하기

n8n은 여러 서비스와 API를 연결해 자동화를 만드는 도구입니다. 플랜과 기능은 바뀔 수 있으므로 가입 전 공식 사이트에서 현재 조건을 확인하고, API 키는 n8n Credentials에 저장합니다.

n8n 설정 방법 안내 요청

n8n을 처음 시작하는 방법을 알려줘.

내 상황:
- 컴퓨터: [Mac / Windows]
- 목표: Vercel에 배포된 웹사이트 콘텐츠를 자동 업데이트
- 데이터 소스: [예: RSS 피드 / YouTube / 뉴스 API]
- 업데이트 주기: [예: 매일 오전 9시]

다음 순서로 설명해줘:

1. n8n 클라우드 또는 셀프호스팅 선택 기준
2. 첫 워크플로우 화면 구조
3. 핵심 노드 3가지 설명:
   Schedule Trigger, HTTP Request, Set 또는 Code 노드
4. 내 자동화에 필요한 노드 목록
5. API 키를 Credentials에 저장하는 방법
6. 테스트 실행과 에러 확인 방법

코딩 비전공자도 따라할 수 있게 설명해줘.

05. 핵심 자동화 워크플로우 3종

Advanced 포트폴리오에서 가장 설득력 있는 자동화는 실제 운영 흐름입니다. 콘텐츠 자동 업데이트, YouTube 영상 연동, 폼 제출 자동 처리를 기본 3종으로 설계합니다.

워크플로우 01. 뉴스/콘텐츠 자동 업데이트

내 웹사이트에 뉴스/콘텐츠를 매일 자동 업데이트하는 n8n 워크플로우를 설계해줘.

설정 정보:
- 데이터 소스: [RSS URL 또는 API 엔드포인트]
- 수집 항목: [예: 제목, 링크, 날짜, 썸네일]
- Claude 가공: [예: 3줄 요약 + 한국어 번역 + 태그 3개 추출]
- 저장 파일: [예: /data/news.json]
- GitHub 저장소: [저장소 이름]
- 실행 주기: [예: 매일 오전 9시]
- 에러 알림: [Slack / Gmail]

다음 형식으로 설계해줘:
1. Schedule Trigger: cron 설정값
2. HTTP Request: URL, method, headers
3. Code Node: 데이터 파싱 JS 코드
4. Claude AI: system prompt + user prompt 설정
5. Set Node: JSON 구조 정의
6. GitHub: 파일 경로, 커밋 메시지 형식
7. Error Trigger: 실패 시 알림 설정

각 노드의 실제 설정값을 n8n에 입력할 수 있게 구체적으로 써줘.

워크플로우 02. YouTube 영상 자동 연동

YouTube 채널 새 영상을 자동으로 웹사이트에 추가하는 n8n 워크플로우를 설계해줘.

설정 정보:
- YouTube 채널 ID: [채널 ID]
- API 키: [YouTube Data API v3 키. n8n Credential로 저장]
- 체크 주기: [예: 3시간마다]
- Claude 처리: [예: 제목 한국어화 + 카테고리 자동 분류 + 추천 이유 1줄]
- 저장 파일: [예: /data/videos.json]
- 중복 방지: 이미 추가된 videoId는 스킵

노드 설계:
1. Schedule Trigger 설정
2. YouTube API 호출: 최신 10개
3. IF Node: 신규 영상만 통과
4. Claude AI: 메타데이터 가공
5. Read/Write JSON: 기존 파일 로드 → 신규 항목 추가 → 저장
6. GitHub Push

중복 체크 로직을 JS 코드로 작성해줘.
기존 videos.json의 videoId 배열과 비교하는 방식으로.

워크플로우 03. 폼 제출 자동 처리

웹사이트 폼 제출을 자동으로 처리하는 n8n 워크플로우를 설계해줘.

설정 정보:
- 폼 유형: [예: 강의 수강 신청 / 포트폴리오 피드백 문의]
- 폼 필드: [이름, 이메일, 내용 등]
- Claude 처리: [문의 유형 분류 + 자동 답변 초안 생성]
- 발송 이메일: [Gmail 계정]
- 저장 시트: [Google Sheets 이름]
- 관리자 알림: [Slack 채널명]

노드 설계:
1. Webhook: URL 생성 + 폼과 연결 방법
2. Claude AI: 분류 prompt + 답변 생성 prompt
3. Gmail: 신청자 확인 메일 템플릿
4. Google Sheets: 헤더 구조 + 행 추가 설정
5. Slack: 메시지 포맷

Claude AI 노드의 system prompt와 user prompt를 실제 사용 가능한 형태로 작성해줘.
Gmail 자동 발송 메일 본문도 한국어로 완성해줘.

06. n8n + Claude API 연동

n8n 안에서 Claude를 호출할 때는 내장 Anthropic/Claude 노드 또는 HTTP Request 노드를 사용할 수 있습니다. 모델명과 API 형식은 바뀔 수 있으므로 실제 적용 전 공식 문서를 확인하고, 출력은 파싱 가능한 JSON으로 제한하는 것이 좋습니다.

n8n Claude 노드 설정 프롬프트

n8n 워크플로우에서 Claude API를 연동하는 방법을 알려줘.

내 상황:
- n8n 버전: [클라우드 / 셀프호스팅]
- 사용 목적: [예: 뉴스 요약 생성]
- 입력 데이터: [이전 노드에서 오는 데이터 구조 설명]
- 원하는 출력: [예: JSON 형태의 요약 + 태그]

다음 두 가지 방법을 모두 알려줘:

방법 A: Anthropic 또는 Claude 내장 노드 사용
- Credentials 등록 방법
- 모델 선택 기준
- System Prompt 설정
- 이전 노드 데이터를 User Message에 넣는 방법

방법 B: HTTP Request 노드로 직접 호출
- URL, Method, Headers 설정
- Body JSON 구조
- 응답 데이터 파싱 방법

출력이 반드시 파싱 가능한 JSON으로 나오도록 System Prompt와 User Prompt를 작성해줘.
API 키는 코드에 직접 쓰지 않고 Credentials에 저장하는 방식으로 설명해줘.

HTTP Request Body 예시

{
  "model": "[현재 사용 가능한 Claude 모델명]",
  "max_tokens": 1000,
  "system": "너는 콘텐츠 큐레이터야. 입력된 뉴스 기사를 분석해서 반드시 JSON만 반환해줘. 다른 텍스트는 포함하지 마.",
  "messages": [
    {
      "role": "user",
      "content": "다음 기사를 분석해줘:\\n제목: {{ $json.title }}\\n내용: {{ $json.description }}\\n\\n다음 JSON 형식으로만 반환해줘:\\n{\"summary\": \"3줄 요약\", \"tags\": [\"태그1\", \"태그2\"], \"category\": \"분류\"}"
    }
  ]
}

07. GitHub Actions 자동 배포

n8n이 JSON 파일을 GitHub에 커밋하면 GitHub Actions가 JSON을 검증하고 Vercel 배포를 실행합니다. Secrets에는 Vercel 토큰과 프로젝트 ID를 저장하고, 실패 시 알림이 오도록 설계합니다.

GitHub Actions .yml 생성

내 프로젝트에 맞는 GitHub Actions .yml 파일을 만들어줘.

프로젝트 정보:
- 저장소: [GitHub 저장소 이름]
- 트리거: data/ 폴더 내 JSON 파일이 push될 때
- 빌드: [순수 HTML/CSS/JS — 빌드 없음 / 또는 빌드 명령어]
- 배포: Vercel CLI 사용
- 브랜치: main

다음을 포함한 완성된 .yml 파일을 만들어줘:
1. 트리거 설정
   - push 이벤트, paths: ['data/**']
   - 수동 실행 workflow_dispatch 포함
2. 작업 단계
   - checkout
   - JSON 유효성 검증
   - Vercel 배포. VERCEL_TOKEN Secret 사용
   - 배포 성공/실패 Slack 알림
3. 필요한 GitHub Secrets 이름과 값 설명

파일 경로는 .github/workflows/auto-deploy.yml 로 안내해줘.
주석을 한국어로 달아줘.

auto-deploy.yml 예시

name: Auto Deploy on Data Update

on:
  push:
    branches: [main]
    paths:
      - 'data/**'
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: 저장소 체크아웃
        uses: actions/checkout@v4

      - name: JSON 유효성 검증
        run: |
          for file in data/*.json; do
            python3 -c "import json; json.load(open('$file'))"
            echo "$file 검증 통과"
          done

      - name: Vercel 배포
        env:
          VERCEL_TOKEN: ${{ secrets.VERCEL_TOKEN }}
          VERCEL_ORG_ID: ${{ secrets.VERCEL_ORG_ID }}
          VERCEL_PROJECT_ID: ${{ secrets.VERCEL_PROJECT_ID }}
        run: |
          npm install -g vercel
          vercel --prod --token=$VERCEL_TOKEN
