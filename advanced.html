AI Automatic · 자동화 도구

여러가지 다양한AI자동화 도구

이 페이지는 첨부 자료의 상세 설명을 유지하면서, JYOUNG UX LAB의 기존 레이아웃 안에서 다시 정리한 자동화 도구 가이드입니다. 각 도구는 개별적으로 사용할 수 있고, 필요한 경우 프로젝트 흐름에 맞게 차례대로 연결합니다.

읽는 방식 먼저 전체 이해 후 필요한 도구만 선택 도구 n8n · Playwright · Pillow · FFmpeg · Remotion · MCP 결과물 자동화 요청 공식과 실전 프롬프트
읽는 방법 n8n Playwright Pillow / OpenCV FFmpeg Remotion 파일 자동화 MCP 요청 공식

이 페이지는 순서대로 모두 실행하는 강의가 아니라, 필요한 자동화 도구를 골라 쓰는 사용 설명서입니다.

앞 페이지의 Harness가 “AI 팀을 어떤 구조로 운영할지”를 다뤘다면, 이 페이지는 실제 작업에서 쓰는 자동화 도구를 하나씩 이해하는 자료입니다. n8n, Playwright, Pillow, FFmpeg, Remotion, 파일 자동화, MCP는 각각 따로 사용할 수 있고, 프로젝트가 커질 때 서로 연결해서 하나의 자동화 흐름으로 만들 수 있습니다.

  • 처음 읽을 때: n8n부터 요청 공식까지 위에서 아래로 한 번 읽어 도구의 역할을 이해합니다.
  • 실제 적용할 때: 지금 필요한 작업에 맞는 도구만 골라 프롬프트 예시를 바꿔 사용합니다.
  • 포트폴리오에 넣을 때: 어떤 문제를 자동화했고, 어떤 도구를 선택했고, 결과물이 어디에 저장됐는지 기록합니다.
  • 여러 도구를 연결할 때: n8n은 흐름을 예약하고 연결하며, 웹 수집은 Playwright, 이미지 처리는 Pillow, 영상 변환은 FFmpeg처럼 역할을 나눠 파이프라인으로 묶습니다.

00. n8n — 자동화 흐름을 연결하는 운영 도구

시간, 데이터, AI 처리, 저장, 알림을 하나의 워크플로우로 묶는 도구

이게 뭔가요?

n8n은 여러 서비스를 선으로 연결해서 반복 작업이 자동으로 실행되게 만드는 워크플로우 도구입니다.
예를 들어 매일 오전 9시에 데이터를 가져오고, Claude로 요약하고, JSON 파일로 저장하고, GitHub에 업데이트한 뒤, 관리자에게 알림을 보내는 흐름을 만들 수 있습니다.

쉽게 말하면?
Playwright, Pillow, FFmpeg가 각각 손과 도구라면, n8n은 언제 어떤 도구를 어떤 순서로 실행할지 정하는 작업 관리자입니다. 그래서 n8n은 혼자 쓰기보다 다른 도구와 연결될 때 더 강력합니다.
n8n은 언제 쓰나요? 사용 기준
정해진 시간에 실행

매일, 매주, 3시간마다처럼 사람이 직접 시작하지 않아도 자동 실행합니다.

여러 서비스를 연결

RSS, YouTube, Claude API, Google Sheets, GitHub, Slack을 한 흐름으로 묶습니다.

조건에 따라 분기

새 데이터만 통과시키거나, 실패하면 알림을 보내는 조건 처리를 만듭니다.

반복 운영 자동화

포트폴리오 콘텐츠 업데이트, 문의 처리, 뉴스 요약, 영상 목록 갱신을 자동화합니다.

이 흐름은 “콘텐츠 자동 업데이트”에 가장 많이 쓰입니다. 포트폴리오 사이트라면 뉴스, 강의 자료, YouTube 영상 목록, 수강생 결과물 목록을 자동 갱신하는 구조로 설명할 수 있습니다.

n8n 워크플로우를 이렇게 요청하세요
"내 웹사이트의 data/news.json을 매일 오전 9시에 자동 업데이트하는 n8n 워크플로우를 설계해줘.
데이터 소스는 [RSS 또는 API URL]이고, Claude로 제목/요약/태그를 만들고,
중복된 URL은 제거한 뒤 GitHub에 커밋하고, 실패하면 Gmail로 알림이 오게 해줘.

적용 순서: 먼저 n8n으로 전체 흐름을 설계하고, 웹 조작이 필요하면 Playwright를 붙이고, 이미지/영상 가공이 필요하면 Pillow나 FFmpeg를 뒤에 연결합니다.

01. Playwright — 웹 자동화

웹사이트를 사람 대신 자동으로 조작하는 도구

이게 뭔가요?

Playwright는 컴퓨터가 사람 대신 인터넷 브라우저를 조작하게 해주는 도구입니다.
우리가 손으로 하는 "주소창에 URL 입력 → 버튼 클릭 → 글자 입력 → 결과 복사" 같은 작업을, 코드 한 줄이 자동으로 해줍니다.

쉽게 말하면?
내가 매일 아침 네이버에 들어가서 날씨를 확인하고 메모장에 적는 작업을 한다고 해봐요. Playwright는 "나 대신 이 작업을 자동으로 해주는 로봇 손"이에요. 클릭도 하고, 글도 쓰고, 내용도 긁어와요.
할 수 있는 것들 기능 목록
정보 수집 (크롤링)

쇼핑몰 가격, 뉴스 제목, 블로그 글 등 원하는 정보를 자동으로 모아옴

자동 입력

폼 작성, 검색어 입력, 로그인 자동화 등 반복 입력 작업 처리

화면 캡처

특정 웹페이지를 PNG/PDF로 자동 저장. 포트폴리오 스크린샷도 가능

데이터 저장

수집한 정보를 CSV, JSON 파일로 자동 정리해서 저장

// 1단계: 브라우저를 열고 네이버로 이동
await page.goto('https://www.naver.com');

// 2단계: 검색창을 찾아서 키워드 입력
await page.locator('#query').fill('디자인 트렌드 2025');

// 3단계: 검색 버튼 클릭
await page.locator('button[type=submit]').click();

// 4단계: 결과 제목들 모두 가져오기

// 5단계: 화면 캡처해서 PNG로 저장
await page.screenshot({ path: 'results/result.png' });

02. Pillow · OpenCV — 이미지 자동화

이미지를 코드로 만들고, 자르고, 꾸미는 도구

이게 뭔가요?

Pillow는 Python으로 이미지를 다루는 도구예요.
포토샵을 손으로 켜지 않아도, 코드가 알아서 이미지를 자르고, 크기를 바꾸고, 글씨도 넣어줍니다. 카드뉴스 9장을 한 번에 만들 수도 있어요!

쉽게 말하면?
포토샵 작업을 매번 손으로 하는 대신, "이 템플릿으로 카드 10장 자동 생성해줘"라고 하면 Pillow가 알아서 이미지 10장을 뚝딱 만들어내는 거예요.

img.save('outputs/01_카드.png')

03. FFmpeg · MoviePy — 영상 자동화

영상을 자르고, 붙이고, 변환하는 영상 편집 엔진

이게 뭔가요?

FFmpeg는 영상 파일을 다루는 전문 엔진이에요.
프리미어 프로를 열지 않아도 터미널 명령어 한 줄로 영상을 자르고, 합치고, 변환할 수 있어요. 유튜브 쇼츠용 세로 영상 변환, BGM 합치기도 자동으로 됩니다.

쉽게 말하면?
영상 편집 프로그램 없이 "이 영상 앞뒤 1초씩 잘라서 세로 영상으로 바꿔줘"라고 하면 FFmpeg가 몇 초 만에 처리해버려요. 100개 영상도 한 번에 가능해요!
주요 명령어 설명 핵심 코드
명령어 옵션뜻 (쉽게)예시
ffmpeg -i입력 파일 지정하기원본 영상 불러오기
-ss 00:00:05영상의 시작 지점 설정5초 지점부터 자르기
-t 00:00:30영상 길이 설정30초짜리로 만들기
-vf scale=1080:1920영상 해상도 변경세로 영상(릴스) 크기로
-c:v libx264영상 인코딩 방식 설정고품질 MP4 출력
-shortest가장 짧은 트랙 기준으로 자르기BGM 합칠 때 길이 맞추기
실제 코드 예시 세로 영상 변환
ffmpeg — 세로 영상 변환 + 앞뒤 1초 자르기 
# 원본 영상을 세로(1080x1920)로 변환하고
# 앞에서 1초, 뒤에서 1초 잘라내기

ffmpeg -i input.mp4 \
  -vf scale=1080:1920 \
  -ss 00:00:01 \
  -t 00:00:28 \
  -c:v libx264 -c:a aac \
  -y output_세로.mp4

# videos/ 폴더 안의 모든 mp4를 한 번에 변환하려면:
for f in videos/*.mp4; do
  ffmpeg -i "$f" -vf scale=1080:1920 "outputs/${f%.mp4}_세로.mp4"
done
Claude Code에게 이렇게 요청하세요
"videos/ 폴더 안의 mp4 파일을 모두
1080x1920 세로 영상으로 바꾸고,
앞뒤 1초씩 잘라서 outputs/ 폴더에 저장해줘."

옵션 외울 필요 없어요! Claude Code에 "세로 영상으로 변환해줘"라고 말하면 ffmpeg 명령어를 알아서 만들어줘요. 옵션의 의미만 알면 요청이 훨씬 정확해져요.

04. Remotion · HyperFrames — 코드로 영상 만들기

React/HTML 코드를 영상(MP4)으로 출력하는 도구

이게 뭔가요?

Remotion은 웹 개발하듯이 영상을 만드는 도구예요.
HTML과 CSS로 화면을 디자인하면, 그게 그대로 MP4 영상이 됩니다. 애니메이션 효과, 타이틀 카드, 자막 영상 등을 코드로 정밀하게 제어할 수 있어요.

쉽게 말하면?
파워포인트 슬라이드를 만들면 그게 영상이 되는 것과 비슷해요. 카드 5장을 순서대로 보여주는 30초 쇼츠 영상을 코드 한 번 작성으로 뚝딱 만들 수 있어요.
Remotion vs HyperFrames 차이 비교
도구기반언제 쓰나요?
RemotionReact (JSX)React를 아는 사람, 정밀한 애니메이션 제어
HyperFramesHTML/CSS/JSHTML만 알아도 영상 만들기 가능
실제 코드 예시 카드 등장 애니메이션
remotion — 카드 순서대로 등장하는 쇼츠 영상 
import { useCurrentFrame, interpolate, Sequence } from 'remotion';

// 카드 1장 컴포넌트: 프레임에 따라 페이드인 효과
const Card = ({ title, color }) => {
  const frame = useCurrentFrame(); // 현재 몇 번째 프레임인지
  const opacity = interpolate(frame, [0, 15], [0, 1]); // 0→1 페이드인

  return (
    <div style={{ opacity, background: color, padding: '40px' }}>
      <h1>{title}</h1>
    </div>
  );
};

// 카드 5장을 각 90프레임(3초)씩 순서대로 재생

05. 파일 자동화

폴더 안의 파일을 찾고, 이름 바꾸고, 분류하기

이게 뭔가요?

파일 자동화는 컴퓨터 폴더 안을 청소부처럼 정리해주는 기능이에요.
downloads 폴더에 막 쌓인 파일들을 이미지/영상/문서로 자동 분류하고, 파일명에 날짜도 자동으로 붙여줍니다.

쉽게 말하면?
책상 위 뒤섞인 서류를 "이건 이미지, 이건 영상, 이건 문서"로 자동 분류해서 각 폴더로 옮겨주는 정리 로봇이에요.
실제 코드 예시 파일 자동 분류 + 날짜 추가
python — downloads 폴더 자동 정리 
import os, shutil
from datetime import date

today = date.today().strftime("%Y-%m-%d")  # 오늘 날짜: 2025-05-27

# 파일 종류별 폴더 지정
folders = {
  '이미지': ['.jpg', '.png', '.gif', '.webp'],
  '영상':  ['.mp4', '.mov', '.avi'],
  '문서':  ['.pdf', '.docx', '.pptx'],
}

for file in os.listdir('downloads'):
  ext = os.path.splitext(file)[1].lower()  # 확장자 확인

  for folder, exts in folders.items():
    if ext in exts:
      os.makedirs(folder, exist_ok=True)  # 폴더 없으면 생성
      new_name = f"{today}_{file}"          # 날짜 앞에 붙이기
      shutil.move(f"downloads/{file}", f"{folder}/{new_name}")

각 파일명 앞에 오늘 날짜(YYYY-MM-DD)를 붙여줘."

06. MCP — 외부 서비스 연결

Notion, Google Drive, GitHub 등을 Claude Code에 연결하기

이게 뭔가요?

MCP(Model Context Protocol)는 Claude Code가 외부 서비스를 직접 사용할 수 있게 해주는 연결 플러그예요.
Notion에 글을 쓰거나, Google Drive에서 파일을 읽거나, GitHub 이슈를 확인하는 것을 말 한마디로 할 수 있게 됩니다.

쉽게 말하면?
스마트폰에 앱을 설치하면 기능이 늘어나죠? MCP는 Claude Code에 앱을 설치하는 것과 같아요. Notion MCP를 연결하면 Claude가 Notion을 직접 조작할 수 있어요.
MCP 연결 방법 설정 파일
  • 1
    설정 파일 열기 ~/.claude/claude_desktop_config.json 파일을 텍스트 에디터로 열어요
  • 2
    MCP 서버 정보 추가 연결하고 싶은 서비스(Notion, GitHub 등)의 MCP 설정 코드를 붙여넣어요
  • 3
    Claude Code 재시작 설정을 저장하고 Claude Code를 껐다가 다시 켜면 연결 완료!
claude_desktop_config.json — Notion + Playwright MCP 연결 예시 
{
  "mcpServers": {
    // Notion MCP 연결
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server"],
      "env": { "NOTION_API_KEY": "your_notion_key" }
    },
    // Playwright (웹 자동화) MCP 연결
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
MCP 연결 후 이렇게 요청하세요
"오늘 작업 완료 목록을 Notion에 정리해줘.
제목은 '2025-05-27 작업일지'로 하고
완료한 항목 3개를 체크리스트로 추가해줘."

MCP 서비스마다 API 키나 로그인 연동이 필요해요. 각 서비스(Notion, GitHub 등)에서 API 키를 발급받아 설정 파일에 입력해야 합니다.
