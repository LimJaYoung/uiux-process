00. AI 시작하기

원본 가이드의 핵심은 “좋은 질문을 단계별로 쌓아가는 것”입니다. Claude는 긴 문맥 정리에 강하고, ChatGPT는 기획 확장과 대안 비교에 유용하며, Codex는 실제 파일을 수정하고 구현을 이어가는 단계에서 함께 쓰기 좋습니다.

  • Claude: claude.ai
  • ChatGPT: chatgpt.com
  • Codex: 구현 파일 수정, QA, 배포 전 체크에 활용

유료 플랜은 더 긴 대화와 높은 사용량에 도움이 되지만, Beginner MVP 실습은 무료 플랜으로도 시작할 수 있습니다. 단, AI 답변은 초안이므로 최신 정보와 수치는 반드시 검색 자료로 검증하세요.

AI를 쓸 때 지킬 3가지 원칙

  • 역할을 먼저 줍니다. 예: “너는 시니어 UX 디자이너야.”
  • 출력 형식을 지정합니다. 예: “표로 정리해줘”, “번호 목록으로 작성해줘.”
  • 한 번에 완성하려 하지 말고, 대화를 이어가며 좁힙니다.
너는 시니어 UX 디자이너이자 AI 활용 전문가야.

나는 UIUX 디자이너이고, 지금 AI를 활용해서 첫 번째 MVP 포트폴리오를 만들고 있어.
내가 만들 서비스: [아직 미정 / 또는 서비스명 입력]
타깃 사용자: [예: 20대 직장인]
제작 목표: 배포 가능한 실제 작동 URL + 케이스 스터디 문서

이번 대화에서는 [오늘 할 작업 단계]를 진행할 거야.
모든 답변은 한국어로, 초보자도 이해할 수 있게 설명해줘.

01. 아이템 선정

좋은 Beginner MVP는 해결할 문제가 명확하고, 핵심 기능이 1~2개이며, 3~5개 화면 안에 담길 수 있어야 합니다. 처음부터 서비스 전체를 만들려 하지 말고 “작동하는 작은 가치”를 먼저 정합니다.

  • 서비스명 확정
  • 핵심 기능 1~2개
  • MVP 범위 문서

아이디어 20개 뽑기

나는 [직업 / 관심사 / 자주 겪는 불편함]을 가진 사람이야.

다음 조건에 맞는 서비스 아이디어 20개를 제안해줘:
- 핵심 기능이 1~2개인 심플한 서비스
- 코딩 비전공자도 AI로 구현 가능한 수준
- 실제 사람들이 쓸 수 있는 현실적인 서비스
- 모바일 또는 웹 중 하나로 만들 수 있는 것

결과는 다음 형식으로 표로 정리해줘:
| 번호 | 서비스명 | 해결하는 문제 | 핵심 기능 | 타깃 사용자 |

아이템 실현 가능성 검토

위 아이디어 중에서 내가 관심 있는 항목은 [번호들]이야.

각 항목에 대해 아래 기준으로 점수를 매겨줘. 각 항목은 1~5점으로 평가해줘.
1. 문제의 명확성: 해결하려는 불편이 분명한가?
2. MVP 구현 난이도: HTML/CSS/JS 수준에서 만들 수 있나? 점수 높을수록 쉬움
3. 포트폴리오 설명력: 면접이나 포트폴리오에서 설명하기 좋은가?
4. 차별성: 이미 있는 서비스와 다른 점이 있나?

합산 점수가 높은 순으로 TOP 3를 추천해주고, 각각의 이유를 2~3줄로 설명해줘.

02. 리서치 조사와 분석

AI의 리서치 답변은 케이스 스터디 초안을 빠르게 만드는 데 좋지만, 최신 수치와 실제 경쟁 서비스 정보는 Google Trends, 앱스토어 리뷰, 서비스 공식 페이지, 통계 자료로 검증해야 합니다.

  • 시장 현황 요약
  • 경쟁 서비스 비교표
  • 사용자 인사이트

시장 현황 파악

[서비스명]이 속한 시장에 대해 알려줘.

다음 항목을 각각 2~3문장으로 요약해줘:

1. 이 시장을 사용하는 주요 사용자 유형: 나이대, 직업, 상황
2. 최근 트렌드: 어떤 방향으로 변화하고 있나?
3. 사용자들이 현재 가장 불편해하는 점 3가지
4. 기존 해결책의 한계: 왜 아직도 불편한가?

포트폴리오 케이스 스터디에 쓸 수 있게 근거 문장 형식으로 작성해줘.
마지막에는 실제 검색으로 검증해야 할 항목도 따로 표시해줘.

경쟁 서비스 비교표

[서비스명]과 유사한 기존 서비스 3~5개를 정리해줘.

다음 항목을 표로 정리해줘:
| 서비스명 | 핵심 기능 | 주요 사용자 | 강점 | 약점 | 가격 또는 이용 조건 |

분석 후 다음 질문에도 답해줘:
- 이 서비스들이 공통적으로 해결하지 못한 불편은 무엇인가?
- 내가 만드는 서비스가 비집고 들어갈 수 있는 틈새는 어디인가?
- 실제 검색으로 다시 확인해야 할 정보는 무엇인가?

가상 사용자 인터뷰

[서비스명]의 타깃 사용자 [예: 20대 프리랜서 디자이너] 3명을 가상으로 인터뷰해줘.

각 인물은 달라야 해:
- 인물 1: 이 문제를 가장 심하게 겪는 사람
- 인물 2: 기존 서비스를 쓰다가 불만을 느낀 사람
- 인물 3: 아예 해결을 포기한 사람

각 인물에 대해 다음 형식으로 작성해줘:
이름 / 나이 / 직업
인터뷰 주요 발언 2~3문장, 직접 화법
핵심 불편 한 줄 요약
이 사람에게 필요한 것 한 줄 요약

주의: 이 내용은 실제 인터뷰가 아니라 가설 리서치로 표시할 수 있게 작성해줘.

03. 문제 정의

문제 정의는 “누가, 어떤 상황에서, 왜 불편한가”를 분명하게 만드는 단계입니다. “앱이 불편하다”가 아니라 “20대 직장인이 퇴근 후 자기계발 계획을 세울 때, 목표가 추상적이라 지속하지 못한다”처럼 구체화합니다.

  • HMW 문장
  • 문제 정의서 1page
  • 페르소나

HMW 문제 정의문 작성

앞에서 분석한 내용을 바탕으로 HMW 문제 정의문을 작성해줘.

형식:
"우리는 어떻게 하면 [타깃 사용자]가 [상황]에서 [핵심 불편]을 해결할 수 있을까?"

조건:
- 특정 솔루션을 전제하지 않을 것. 예: "앱을 만들어서"라고 쓰지 말 것
- 너무 넓지도, 너무 좁지도 않은 범위로 작성할 것
- 3가지 버전을 작성하고, 각각의 포커스가 달라야 함

작성 후, 3가지 중 포트폴리오 케이스 스터디에 가장 적합한 것을 1개 추천하고 이유를 설명해줘.

문제 정의서 완성

지금까지 정리된 내용을 바탕으로 문제 정의서를 작성해줘.

다음 형식으로 작성해줘:

## 문제 정의서: [서비스명]

### 1. 배경
시장 상황 2~3줄

### 2. 타깃 사용자
페르소나 1명: 이름, 나이, 직업, 하루 루틴, 핵심 불편

### 3. 핵심 문제
HMW 문장 1개

### 4. 기존 해결책의 한계
경쟁 서비스들이 왜 이 문제를 해결하지 못했는지 2~3줄

### 5. 우리의 기회
우리 서비스가 만들 수 있는 차이 2~3줄

포트폴리오에 바로 붙여 쓸 수 있게 간결하고 논리적으로 써줘.

04. 솔루션 제시

문제를 정했다면 바로 화면을 만들지 말고, 어떤 방식으로 해결할지 여러 선택지를 만든 뒤 MVP에 맞는 솔루션을 고릅니다.

  • 핵심 가치 제안문
  • 솔루션 요약 카드
  • MVP 기능 확정

솔루션 아이디어 발산

문제 정의서를 바탕으로 [서비스명]의 솔루션 아이디어를 발산해줘.

제약 없이 10가지 해결 방식을 제안해줘.
각 아이디어는 다음 형식으로 작성해줘:
- 아이디어명: 한 줄 설명
- 이 방식이 문제를 해결하는 원리: 2줄
- MVP에서 구현 가능성: 상/중/하

10가지 작성 후, MVP 단계에서 현실적으로 구현 가능한 것 TOP 3를 골라줘.

핵심 가치 제안

선택한 솔루션을 바탕으로 [서비스명]의 핵심 가치 제안문을 써줘.

형식:
"[타깃 사용자]를 위한 [무엇]. [기존 방식]과 달리 [차별점]으로 [결과]를 만들어냅니다."

3가지 버전으로 작성해줘:
- 버전 A: 기능 중심, 무엇을 할 수 있나
- 버전 B: 감성 중심, 어떤 기분이나 변화를 주나
- 버전 C: 포트폴리오 발표용, 면접관에게 어필하는 버전

각 버전은 2문장 이내로 써줘.

솔루션 요약 카드

다음 형식으로 솔루션 요약 카드를 작성해줘.

문제 Problem:
한 문장으로

솔루션 Solution:
한 문장으로

핵심 차별점 Key Differentiator:
기존 서비스와 다른 점 1가지

타깃 사용자:
구체적인 한 줄

기대 효과:
이 서비스를 쓰면 사용자에게 어떤 변화가 생기나? 한 줄

간결하게, 포트폴리오에 그대로 붙여넣을 수 있도록 써줘.

06. AI 구현 요청

구현 단계에서는 Claude나 ChatGPT로 화면 초안을 만들고, 실제 프로젝트 파일 수정과 반복 QA는 Codex와 함께 진행하면 안정적입니다. 화면 명세서와 디자인 방향이 없으면 AI가 엉뚱한 결과를 만들기 쉬우니 먼저 기준을 고정하세요.

  • HTML/CSS 파일
  • 디자인 시스템 CSS
  • 작동하는 화면
