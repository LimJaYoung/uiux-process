언제 쓰나

리서치 메모, 회의 기록, 경쟁 분석, 화면 요구사항처럼 긴 입력을 하나의 UX 문서로 묶어야 할 때 좋습니다.

활용 방법

  • 사용자 여정을 단계별로 정리합니다.
  • 화면 목록과 상태 화면을 빠짐없이 점검합니다.
  • 디자인 의사결정의 이유를 면접 답변처럼 다듬습니다.
  • 케이스 스터디의 시작, 중간, 끝 구조를 잡습니다.

바로 쓰는 프롬프트

당신은 시니어 UX 디자이너이자 포트폴리오 멘토입니다.
아래 기획 자료를 바탕으로 UX 설계 문서를 정리해주세요.

1. 핵심 사용자 목표
2. 사용자 여정 단계
3. 필요한 화면 목록
4. 각 화면의 목적과 핵심 콘텐츠
5. 빈 상태, 오류 상태, 성공 상태
6. 포트폴리오에서 설명할 의사결정 이유
